## Objective

Every `make` target and every CI job does real work and passes on the current
skeleton, locally verifiable on this machine (no Docker available — see
`agents/handoffs/decisions.md` D-002).

## In-scope files

`Makefile`, `.github/workflows/ci.yml`, `.github/workflows/ams-version-matrix.yml`
(structure only), `deploy/docker/pulse.Dockerfile`, `deploy/docker-compose.yml`,
`.gitignore`.

## Work items

1. **Makefile robustness:** `build-web`/`build-sdk`/`test-*`/`lint` targets must
   work in a clean shell: guard `npm install` when `node_modules/` is missing
   (use a sentinel/`node_modules` dependency, not unconditional installs).
   Known failure: `make build` died with `sh: tsc: command not found` when
   `web/node_modules` was absent.
2. **ci.yml — replace the three echo-stub jobs with real ones:**
   - `contracts`: validate the 3 event schemas compile as JSON Schema 2020-12
     (`npx ajv-cli compile --spec=draft2020`) and lint the OpenAPI spec
     (`npx @redocly/cli lint contracts/openapi/pulse-api.yaml` — warnings
     allowed, errors fail).
   - `web`: `npm ci && npm run build` (add `npm run lint`/`test` steps only if
     those scripts exist; do not author web code — file gaps in your report).
   - `sdk`: `npm ci && npm run build && npm run size`. NOTE: the size script is
     currently broken by an SDK packaging mismatch — WO-002 (SDK-01) fixes that
     in parallel; your job must simply invoke the scripts.
   - Add a `make`-level entry point where sensible so CI and local runs share
     commands.
3. **Pin Docker images by digest** in `pulse.Dockerfile` and
   `docker-compose.yml` (node:22-alpine, golang:1.24-alpine, alpine:3.21,
   clickhouse/clickhouse-server:24.8). Fetch digests from the registry HTTP API
   with curl (no Docker daemon present). Keep the human-readable tag in a
   comment. Also add a HEALTHCHECK to the runtime image (`wget -qO- localhost:8090/healthz`).
4. **ams-version-matrix.yml:** keep as structured stub but make the step fail
   gracefully with a clear "pending QA-01 wave-1" message instead of silent echo;
   do not implement the matrix tests (QA-01 owns them).

## Acceptance criteria (testable)

- `make build && make test && make lint` exit 0 from a clean checkout state
  (you may rely on network for npm install).
- Contract validation commands from ci.yml, run locally, exit 0.
- `git diff` touches only your scope (`deploy/`, `.github/`, `Makefile`, `.gitignore`).
- No unpinned image references remain in deploy/.

## Out of scope

Application code, contracts content, sdk/web package.json (SDK-01/FE-01 own
those), Helm (wave 2), actually running Docker.
