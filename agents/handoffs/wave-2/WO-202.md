## Work items

1. **Kafka source** (`collector/kafka`): implement the stub — consumer group on
   AMS native producer topics (`server.kafka_brokers`), JSON decode, normalize
   through the same layer as logtail; reconnect with backoff; lag surfaced in
   /healthz component detail. Use a pure-Go client (franz-go or segmentio/kafka-go
   — justify choice). Verification: in-process fake/contract tests only (D-007.5
   — no broker on this machine; document).
2. **Geo enrichment** (`collector/enrichment`): replace NoopResolver with a
   MaxMind-format reader (oschwald/maxminddb-golang), config-driven `.mmdb` path
   (D-007.4: reader only, no DB bundled; absent path ⇒ no-op). `anonymize_ip`
   switch: zero last octet (v4) / last 80 bits (v6) BEFORE lookup+storage, per
   config `beacon.anonymize_ip`. Country + region per contract enrichment block.
3. **Device enrichment:** replace NoopParser with a small embedded UA parser
   (uap-go or a minimal own matcher — budget-justify; no network). device, os,
   browser per enrichment block; protocol from event context.
4. **Viewer sessions:** stitch viewer_join/leave + beacon heartbeats (BE-02
   feeds beacon events through `domain.EventSink` — same pipeline) into
   `viewer_sessions` rows (ReplacingMergeTree contract): session_id, stream,
   app, start/end, watch_ms, geo/device dims, protocol. Handle absent leave
   events (timeout close after configurable idle, default 5 min).
5. **Ingest health (F4):** extract per-publisher metrics from analytics-log
   events + REST WebRTC client stats: bitrate, fps, keyframe interval, packet
   loss, jitter → `ingest_stats` domain events (contract) + live-aggregator
   per-publisher state with drop detection (bitrate floor breach, fps collapse,
   source gone). Health score: documented, reproducible formula in code comment
   + report (weighted: bitrate stability, fps vs target, keyframe regularity,
   loss/jitter) — BE-02 exposes it via API, FE displays it; the formula doc is
   the contract between you.
   Budget: degradation visible in live state ≤15 s of the source change (test
   with stub source; measure, report).
6. **Fleet (F7)** (`internal/cluster`): periodic cluster-node discovery via
   amsclient ClusterNodes (default 30 s); register/refresh nodes in live
   aggregator (role origin/edge, status, last_seen, version, load); node
   up/down domain events for the alert evaluator; aggregate metrics dedup
   origin/edge double counting (PRD F7 — viewers counted at edge only when a
   stream is served via edges; document the rule). Budget: new node visible
   ≤2 min (test with mock; measure).
7. **Rollup correctness:** ensure raw writes carry the geo/device/protocol dims
   the contract rollup MVs aggregate; integration test (build tag) verifying
   rollup_audience_1h and rollup_qoe_1h populate from inserted raw events.
