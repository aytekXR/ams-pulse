## Objective

Usage/billing accounting that reconciles within ±1%, scheduled statement
generation under 60 s, tenant mapping, CSV/PDF outputs with white-label header,
and scheduled CSV-to-S3 export.

## In-scope files

`server/internal/{reports,query,api,store/meta,config}/`, `server/cmd/pulse`
(assembly, declare — D-005), `server/go.mod`, `go.sum`.

## Work items

1. **Usage accounting** (`internal/reports`): per app/stream/tenant over a
   period: viewer-minutes (from viewer_sessions watch_ms), peak concurrency
   (windowed max from rollups), egress GB — delivered-bytes events where
   available, else bitrate×watch-time model, with `method` field disclosed per
   row (PRD F6 wording), recording GB (recording_ready events). Source:
   rollup_usage_1d + raw fallback for partial days.
2. **Tenant mapping (F6):** tenants table (meta) with stream-name glob pattern
   AND/OR metadata-tag mapping; resolution precedence documented; unmatched
   streams roll into an implicit "unassigned" tenant; CRUD per OpenAPI.
3. **Statements:** monthly (and arbitrary-range) statement generator: CSV
   always; PDF via a pure-Go lib (gofpdf/fpdf successor — justify; CGO=0);
   optional white-label header block (logo path/name/address from
   report_schedules config JSON). Generation budget: <60 s for a month at
   ICP-B scale — prove with seeded synthetic month (report the measured time
   and dataset size).
4. **Reconciliation guarantee (±1%):** a reports-internal verification mode
   that recomputes one statement's viewer-minutes directly from raw
   viewer_sessions/server_events and asserts ≤1% drift vs the rollup-based
   figures — exposed as a Go test AND a `pulse diag --reconcile` flag QA can
   run (document; declare cmd edit).
5. **Schedules:** report_schedules (cron via the WO-203 cron parser, scope,
   tenant, format csv|pdf, destination download|s3) executed by a scheduler
   loop in serve; generated artifacts stored under a configurable dir,
   listed/downloadable per OpenAPI; failure → alert-history entry (severity
   info) + log.
6. **CSV-to-S3 export (F8):** S3-compatible PUT (minio-go or AWS sigv4 by
   hand — justify on dep weight); config: endpoint, bucket, prefix, creds via
   env refs (never stored plaintext — meta encrypted fields); wire as schedule
   destination; retries + alert on repeated failure.
7. **Gating:** reports + schedules + S3 export = Business tier (WO-203's
   entitlement layer); white-label header = Business per PRD §7.12 integrator
   channel (Enterprise white-label PDF polish is Phase 3 — note in report).
8. **Assembly (D-005):** scheduler into serve; declare cmd edits.

## Acceptance criteria (run them; report measured numbers)

- `CGO_ENABLED=0 go build ./... && go vet ./... && go test ./...` green;
  kin-openapi conformance on report endpoints.
- Seeded-month test: known synthetic truth (N sessions, fixed bitrates) →
  statement figures within ±1% of truth; measured generation time <60 s;
  egress `method` field present per row.
- Tenant mapping tests: glob + tag precedence, unassigned fallback.
- Schedule test (fake clock): cron fires, artifact produced, listed via API;
  S3 path exercised against an httptest S3 fake (signature verified).
- `pulse diag --reconcile` runs against the local stack and prints drift.
