## Work items

1. **Beacon ingest** (`collector/beacon` — ARCHITECTURE §3.5 hostile-input
   rules): `POST /ingest/beacon` per OpenAPI on the dedicated ingest listener
   (`server.ingest_listen`, wire `PULSE_INGEST_LISTEN_ADDR` — closes a DOC-01
   wave-1 gap): `X-Pulse-Ingest-Token` auth against ingest_tokens (constant-time
   compare, hashed at rest); per-token rate limit (token bucket, configurable
   default 100 req/s) → 429; body cap (64 KB) → 413; strict schema validation
   against `beacon-event.schema.json` → 422 with error envelope; 202 + async
   write through `domain.EventSink` (enrichment runs in BE-01's pipeline);
   CORS for browser SDKs; never echo tokens.
2. **QoE query surface (F3/F4):** implement the OpenAPI qoe endpoints: summary
   (startup p50/p95, rebuffer ratio, error rate, bitrate timeline — sliceable
   by stream/app/geo/device/time) from beacon_events + rollup_qoe_*; ingest
   health endpoints (per-publisher timelines + health score + drop events)
   from BE-01's live state + ClickHouse history.
3. **Fleet API (F7):** `/fleet/nodes` per OpenAPI from cluster state (role,
   status, last_seen, version, per-node load); node up/down alert rule type
   wired to BE-01's node events.
4. **Alert completion (F5 full):** channels Telegram (bot API), PagerDuty
   (Events API v2), generic webhook (HMAC-SHA256 signature header, documented)
   — closes G1; new rule types: rebuffer_ratio, error_rate, ingest_bitrate_floor
   (from QoE/ingest aggregates), cert_expiry (TLS check of configured AMS/base
   URLs, daily tick); cron-parsed maintenance windows (closes G2); default rule
   pack seeded enabled-but-muted on bootstrap (closes G8).
5. **Prometheus (F8)** (`api`): `GET /metrics` exposition per contract ruling
   (unauthenticated default + optional `PULSE_METRICS_TOKEN`): gauges/counters
   only, bounded label cardinality (app, node — never stream/session):
   concurrent viewers, publishers, per-node cpu/mem, collector lag, ingest
   accept/reject counters, alert states. Use prometheus/client_golang or
   hand-rolled exposition — justify (binary size, CGO=0). Closes G4.
6. **Tier gating (license, §7.11):** entitlement checks at the API/feature
   seams: Free = 1 node, 7-day retention cap, email channel only; Pro adds
   beacons/QoE, Slack+Telegram, 90-day retention, CSV export; Business adds
   PagerDuty+webhook channels, reports (WO-204), multi-tenant, API tokens +
   Prometheus. Fail open for reading already-collected data, closed for gated
   features (ARCHITECTURE §6). Tests per tier boundary.
7. **Carried fixes:** bcrypt for user passwords via golang.org/x/crypto/bcrypt
   (pure Go, CGO=0 — closes G3); CSV export on audience endpoints (closes G5,
   `format=csv` per spec).
8. **Assembly (D-005):** wire ingest listener, channels, Prometheus, gating
