## Objective

The JS QoE beacon SDK, production-real: wraps AMS WebRTC playback and HLS players,
emits schema-conformant beacon events, stays under 15 KB gzipped, and no-ops
gracefully when the collector is unreachable.

## In-scope files

`sdk/` (everything).

## Work items

1. **Core (`src/`):** session model (UUID v4 session stitching, one session per
   playback attempt); event batcher (flush every ≤10 s OR 25 events, whichever
   first; `navigator.sendBeacon` with `fetch` keepalive fallback; bounded retry
   queue with localStorage spill, drop-oldest at cap); sampling
   (`sampleRate: 0..1`, decided once per session); config validation; graceful
   no-op: collector unreachable or no token ⇒ zero throws, zero console spam
   (single debug-level note), zero retry storms (exponential backoff, cap).
2. **One-line init** per PRD: `Pulse.init({ingestUrl, token, streamId, app?,
   metadata?, sampleRate?})` returning a session handle with `attachWebRTC()`,
   `attachHls()`, `attachVideoElement()`, `event()` (custom), `dispose()`.
3. **WebRTC adapter integration:** poll `RTCPeerConnection.getStats()` (5 s
   cadence) → bitrate_change, resolution_change, heartbeat (watch_ms, bitrate,
   dropped_frames); detect startup_complete (first frames rendered) and stalls
   via frame-counter deltas.
4. **HLS/media-element integration:** hls.js events (MANIFEST_LOADED→first
   FRAG_BUFFERED for startup; BUFFER_STALLED/waiting→playing pairs for
   rebuffer_start/end with duration_ms; ERROR with code/fatal) and plain
   `<video>` element events (waiting/playing/stalled/error/ratechange) for
   video.js/native — video.js integration documented via its underlying media
   element, no video.js dependency.
5. **Event mapping:** every emitted event validates against
   `beacon-event.schema.json` (CMCD-aligned names per contract). Round-trip
   tests: emit each event type → `ajv` validate against schema + fixtures.
6. **Size + quality gates:** `npm run size` stays green (15 KB limit) — tree-shakeable,
   zero runtime deps; fix the wave-0 carried gap: add `eslint.config.js` (flat)
   + `lint` script + a real `test` script (vitest + jsdom); typed public API
   (`.d.ts` emitted).
7. **README:** integration guide for AMS JS SDK, hls.js, video.js, plain video
   element; config reference; what is collected (privacy note, anonymize-IP is
   server-side).

## Acceptance criteria (run them; report measured numbers)

- `npm run build && npm run size && npm run lint && npm run test` all green;
  report the measured gzip size.
- Schema round-trip: every event type validates; malformed-config tests prove
  no-throw behavior.
- Unreachable-collector test: requests fail → SDK keeps a bounded queue, backs
  off, player callbacks unaffected (fake timers).
- Batching test: events buffered and flushed ≤10 s; flush-on-`visibilitychange`/
  `pagehide` via sendBeacon.
- No network calls when `sampleRate` excludes the session (test).
