## Work items

1. **F9 baselines** (`internal/anomaly` + `store/meta`): rolling per-(metric, scope,
   window_s) statistics in the frozen `anomaly_baselines` table (mean, stddev,
   sample_count, last_updated). Update incrementally (Welford or EWMA — justify) on
   a periodic tick from the same metric snapshot the alert evaluator already reads
   (viewers, error_rate, rebuffer_ratio at minimum). Upsert keyed by the unique
   index `(metric, scope, window_s)`.
2. **F9 detection + API:** implement `GET /anomalies` per the frozen contract —
   compute AnomalyFlags on read: for each baseline with `sample_count ≥ minSamples`,
   compare the latest observed value; flag when `|observed − mean| / stddev ≥ sigma`
   (default sigma from config, query-overridable per the spec). Return
   {id, metric, scope, observed, expected=mean, sigma=actual-z, ts}. **Calibrate for
   the PRD target (<1 false alarm/node-week):** default sigma (≈3), a minimum sample
   count, and hysteresis/cooldown so steady noise does not flag. Document the
   sensitivity math in code + report (it is the QA-04 test target).
3. **F10 ProbeConfigSource impl** (`store/meta` + glue): implement BE-01's
   `domain.ProbeConfigSource` over the meta `probes` table — `ListEnabled()` reads
   enabled probe rows; `RecordResult()` updates `last_result_id/last_success/
   last_run_at`. This is what makes BE-01's runner live.
4. **F10 probe CRUD** (`api` + `store/meta`): `POST/GET /probes`, `GET/PUT/DELETE
   /probes/{id}` per OpenAPI over the `probes` table (validate `interval_s ≥ 30`,
   protocol enum, URL). `GET /probes/{id}/results` reads BE-01's
   `QueryProbeResults` (via `query`) and returns ProbeResult rows; `GET /probes`
   includes each probe's `last_result` summary from the denorm fields.
5. **Tier gating + assembly:** anomaly detection + probes are Pro+/Business per
   §7.11 (anomaly detection is Enterprise per the PRD pricing table — gate
   `/anomalies` to Enterprise, probes to Pro+; confirm against the wave-2
   entitlement layer and document the matrix). Wire the anomaly tick, the
   ProbeConfigSource, and the runner construction into `pulse serve` together with
   BE-01's hook (declare cmd edits).
