## Objective

App shell + the three wave-1 surfaces: live ops dashboard (F1), basic audience
analytics (F2), alerts management (F5), plus the settings/onboarding flow that
carries the 15-minute-install promise.

## In-scope files

`web/` (everything).

## Work items

1. **Types + client:** add `openapi-typescript` devDep; npm script `generate:api`
   per `contracts/README.md`; `src/api/client.ts`: thin fetch wrapper (bearer token
   from localStorage, JSON/error envelope handling, typed by generated types) +
   `LiveSocket` class for `/live/ws` (auto-reconnect with backoff, snapshot/delta/
   heartbeat envelope per spec).
2. **Shell:** React Router routes per App.tsx comment map (`/`, `/analytics`,
   `/qoe`, `/ingest`, `/alerts`, `/reports`, `/fleet`, `/settings`); left nav +
   header (connection status, license tier badge); auth gate (token entry screen
   when 401; store token); error + loading + empty states as shared components.
   Wave-2 routes render structured "coming in this build" placeholders.
   No external fonts/CDNs (LAN rule). Dark, ops-grade default theme; system font
   stack. Charting: add `recharts` (decision: smallest learning curve, SVG fine
   at our cardinality — note in report if you choose differently and why).
3. **Live dashboard (F1):** overview stat cards (concurrent viewers total/per-app,
   active publishers, node CPU/RAM, protocol mix donut); streams table —
   VIRTUALIZED (500+ rows budget): stream, app, node, state, viewers, bitrate,
   health; sparkline per stream optional; live updates via LiveSocket with REST
   fallback polling; "new stream appears ≤10 s" — driven by server push, just
   don't debounce it away.
4. **Analytics (F2-core):** date-range picker (presets: 24h/7d/30d/custom);
   audience timeseries chart (views, uniques, watch time, peak concurrency);
   totals row; per-stream/app filter; CSV export button hitting the spec'd
   export; geo/device tabs show real API data if present, else honest empty state
   (full F2 lands wave 2).
5. **Alerts (F5):** rules list + create/edit form (metric, operator, threshold,
   window, scope, severity, cooldown, maintenance windows); channels list +
   create (email, Slack) + TEST-FIRE button with result toast; history view
   (state, severity, timeline, dedup/group display); muted/default-pack badge.
6. **Settings/onboarding:** first-run wizard: welcome → add AMS source (REST url,
   credential env-var name guidance, optional log path/webhook secret) → verify
   connection (uses admin source-test endpoint if spec'd, else create+status) →
   ingest/api tokens view → license key entry + tier display. Editable settings
   pages for the same after first run.

## Acceptance criteria

- `npm run build` green (tsc strict); `npm run generate:api` produces types and
  build consumes them (no hand-rolled API shapes — CI-checkable: `git grep`
  proof in report).
- Component tests (vitest + testing-library): LiveSocket reconnect/fallback,
  alerts form validation, virtualized table renders 500 rows under budget
  (assert row virtualization, not 500 DOM nodes).
- Add `lint` and `test` scripts (eslint flat config) so CI's web job has them.
- Mock-server dev story: document (README in web/) how FE runs against `pulse
  serve` or against fixtures.
- Screenshots of each surface in the completion report (use the seeded fixture
  approach you document; static HTML render acceptable).
