# WO-102 — Wave 1 data plane (BE-01)

**Issued by:** ORCH-00 · 2026-06-11
**Assigned:** BE-01 (charter: `agents/definitions/BE-01-backend-dataplane.md`)
**PRD refs:** F1, F2-core technical notes; §7.10 data flow
**Contracts (frozen):** `events/ams-server-event.schema.json`, `db/clickhouse/0001_init.sql`

## Objective

Real, normalized AMS data flowing into ClickHouse and into in-memory live
aggregates, within PRD latency budgets, via a runnable `pulse` binary.

## In-scope files

`server/pkg/amsclient/`, `server/internal/collector/` (NOT `collector/beacon` —
BE-02 wave 2), `server/internal/store/clickhouse/`, `server/internal/domain/`,
`server/cmd/pulse/`, `server/go.mod`, `server/go.sum`. New test files within scope.

## Work items

1. **domain types** (`internal/domain/types.go`): field out ServerEvent,
   BeaconEvent, ViewerSession per the frozen event schemas; AlertRule/Notification
   fields per meta DDL + alert-notification schema (BE-02 consumes — coordinate
   via the contract, not imports of BE-02 code). Add a `LiveSnapshot` type
   (overview totals, per-app, per-stream, per-node) and define interfaces BE-02
   will consume: `domain.LiveProvider` (current snapshot + subscribe-to-updates)
   and `domain.EventSink` (what collectors write into). Keep interfaces here so
   both planes depend only on domain.
2. **amsclient** (`pkg/amsclient`): typed REST v2 client — ListApplications,
   ListBroadcasts (paged), BroadcastStatistics, WebRTCClientStats, ClusterNodes,
   node info; bearer/JWT header auth; timeouts; context; tolerant JSON decoding
   (unknown fields ignored, missing fields zero). NO Pulse-domain knowledge here —
   raw AMS DTOs only.
3. **collector framework** (`internal/collector`): supervisor that runs each
   configured `Source` with restart + exponential backoff; normalization layer
   mapping AMS DTOs/log lines/webhook payloads → `domain.ServerEvent` (the ONLY
   place AMS wire shapes are interpreted, with amsclient); enrichment hooks (geo/
   device fill per contract enrichment block — geo can be a no-op resolver wave 1,
   interface present); dedup (e.g. same publish event from poller + webhook —
   key on type+stream+ts-window); fan-out to EventSink consumers: ClickHouse
   writer, live aggregator.
4. **Sources:** `restpoller` (configurable interval default 5 s; lists broadcasts
   + stats; emits stream_stats, publish start/end transitions, viewer counts,
   node_stats from system endpoints; must surface a new stream ≤10 s — make the
   default interval prove it), `logtail` (tail JSON analytics log; rotation-aware
   — reopen on inode change/truncate; partial-line safe; unknown event types
   skipped + counted), `webhook` (HTTP handler implementing Source; shared-secret
   validation; tolerant parsing across AMS versions). Kafka source is wave 2 —
   leave the stub.
5. **Live aggregator:** in-memory state fed by the event stream: current streams
   (id, app, node, publisher state, viewer counts by protocol, ingest bitrate/fps,
   health score inputs), per-app and total concurrent viewers, per-node health;
   snapshot + subscription API per `domain.LiveProvider`; eviction of stale
   streams (no stats for N intervals → offline transition event).
6. **ClickHouse store** (`internal/store/clickhouse`): clickhouse-go/v2 native
   protocol; batched async inserts (size+time flush, e.g. 1000 events / 2 s);
   write paths for server_events, beacon_events, viewer_sessions; query helpers
   used by BE-02 stay OUT (BE-02's query plane talks ClickHouse itself via a thin
   read interface you expose in domain or store — define `store.Conn` accessor).
   Retry on connect failure at startup (compose race) — bounded, logged.
7. **`pulse` CLI** (`cmd/pulse`): subcommands `serve`, `migrate` (applies
   `contracts/db/clickhouse/*.sql` + delegates meta migration to BE-02's package
   via an assembly hook), `version`, `diag` (basic: config echo w/o secrets,
   connectivity checks). Config loading delegated to `internal/config` (BE-02
   scope — it runs after you; for now call `config.Load` as the stub API defines
   and treat errNotImplemented as fatal-with-message, leaving a clearly marked
   assembly hook comment block for BE-02). Embed web UI static dist via
   `embed.FS` behind a build tag or os.DirFS fallback (document for INFRA).
   Graceful shutdown on SIGINT/SIGTERM.
8. **Migrations runner:** idempotent (schema_migrations table in ClickHouse,
   `${RAW_TTL_DAYS}`/`${ROLLUP_TTL_MONTHS}` substitution per contract mechanism).

## Acceptance criteria (run them; report measured numbers)

- `go build ./... && go vet ./... && go test ./...` green; schema round-trip test:
  collector-normalized events marshal→validate against
  `contracts/events/ams-server-event.schema.json` using fixtures from
  `contracts/events/fixtures/` (use a Go JSON-Schema lib or shell to `npx ajv`
  in a test guarded by network availability — prefer the Go lib).
- Integration test (Go test, build-tagged `integration`): start `/tmp/clickhouse server`
  with temp dir, run migrate, insert 10k synthetic events through the batcher,
  verify counts + TTL clauses exist.
- Latency test with a stub AMS HTTP server: publish appears as domain event in
  ≤10 s with default poll interval (measure, assert, report).
- Logtail test: rotation mid-write, malformed line, unknown type — no crash,
  counters increment.
- Collector survives AMS endpoint going away and returning (backoff test with
  fake clock or short intervals).
