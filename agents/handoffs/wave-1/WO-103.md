## Work items

1. **config** (`internal/config`): parse `deploy/config/pulse.example.yaml`
   surface exactly (server.listen/ingest_listen/base_url; ams.sources[] with env
   credential resolution `PULSE_AMS_<NAME>_TOKEN`; storage.clickhouse_addr/meta/
   postgres_dsn/retention; beacon.sample_rate/anonymize_ip; license.key/
   offline_file). Env overrides (`PULSE_` prefix), validation with actionable
   errors, defaults per example file. `Load(args)` honors `--config` flag.
2. **meta store** (`internal/store/meta`): SQLite via modernc.org/sqlite (pure Go,
   keeps CGO_ENABLED=0); apply `contracts/db/meta/0001_init.sql` idempotently;
   repositories: Users, APITokens, IngestTokens, Sources, AlertRules,
   AlertChannels, AlertHistory, ReportSchedules, Tenants, License. Secrets
   encrypted at rest (AES-GCM, key derived from a `PULSE_SECRET_KEY` env or
   generated-and-stored file — document). Postgres variant: implement DSN switch
   + ping, mark tested-SQLite-only in report (acceptable wave 1).
3. **api** (`internal/api`): chi or stdlib mux router implementing the OpenAPI
   surface for wave 1: /live/*, /analytics/audience (basic), /alerts/*,
   /admin/sources, /admin/tokens, /admin/license, /healthz; bearer auth
   middleware (api_tokens), rate limiting, request logging, CORS for LAN use,
   OpenAPI-conformance test (validate responses against the spec with kin-openapi).
   First-run bootstrap: if no users/tokens exist, `pulse serve` prints a one-time
   admin token (document in report). WS hub `/live/ws`: subscribes to
   LiveProvider, pushes snapshot-then-deltas envelope per spec, heartbeats,
   slow-client eviction. Serve web static dist (BE-01 left the hook).
4. **query** (`internal/query`): live queries from LiveProvider; historical from
   ClickHouse (audience timeseries/totals over raw events wave 1 — rollup reads
   come wave 2); CSV export for audience (`Accept: text/csv` or `format=csv` per
   spec).
5. **alert** (`internal/alert`): evaluator loop on live aggregates (tick ≤5 s):
   rule types stream_offline (stream in rules' scope absent/unpublished for
   threshold window), viewer_drop_pct (drop >X% within Y window), node_cpu /
   node_disk / node_mem threshold; state machine pending→firing→resolved with
   cooldown + group_key dedup (no duplicate storms); maintenance-window
   suppression; alert_history persistence; notifications conforming to
   `alert-notification.schema.json`. Channels (`alert/channels`): email (SMTP,
   net/smtp, STARTTLS) and Slack (incoming webhook); pluggable registry; test-fire
   support (`POST /alerts/channels/{id}/test`). Default rule pack seeded
   enabled-but-muted (PRD F5). Detection→notification must be <30 s by
   construction (tick + cooldown math) — prove with fake-clock test.
6. **license** (`internal/license`): Free tier default (no key): 1 node,
   7-day retention cap, email alerts only; key parsing (signed JSON blob,
   ed25519 — embed a dev public key, document format) unlocking Pro/Business/
   Enterprise entitlements per §7.11; entitlement checks exposed as
   `license.Manager` methods consumed by api/alert/reports; fails open for
   reading already-collected data, closed for gated features (ARCHITECTURE §6).
7. **Assembly (D-005):** extend `cmd/pulse` hooks: wire config → meta store →
   api server + WS + alert evaluator + license manager into `serve`; `migrate`
   gains the meta migration; ensure single-binary serve runs end-to-end.

## Acceptance criteria

- `go build ./... && go vet ./... && go test ./...` green.
- API conformance: handler responses validate against `contracts/openapi/pulse-api.yaml`
  (kin-openapi response validation in httptest).
- Fake-clock alert tests: stream-offline fires within rule window + tick; cooldown
  suppresses repeat ≤cooldown_s; resolved notification sent; maintenance window
  suppresses; storm test (50 streams offline → grouped, not 50 pages... unless
  ungrouped rule) documented behavior.
- End-to-end (httptest + BE-01 fakes): seed LiveProvider fake → /live/overview
  matches; WS client receives snapshot+delta.
- Rules survive restart (SQLite round-trip test).
- Free-tier enforcement test: second node rejected/flagged, retention capped.
